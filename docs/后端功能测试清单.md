# 后端功能测试清单（学情智能预警系统）

> 适用范围：Spring Boot + MyBatis + MySQL 后端。
> 目标：验证“鉴权 -> 业务 -> 算法 -> 预警 -> 干预”主链路可用性与权限边界。

---

## 0. 测试准备

### 0.1 环境前置
- JDK 17+
- Maven 3.9+
- MySQL 8+
- 已执行：
  - `source springboot/src/main/resources/sql/schema.sql`
  - `source springboot/src/main/resources/sql/data.sql`

### 0.2 服务启动
- 后端：`cd springboot && mvn spring-boot:run`
- 默认端口：`http://localhost:9090`

### 0.3 默认账号
- 教师：`t_zhang / 123456`、`t_li / 123456`

## 1. 鉴权与权限测试

### AUTH-01 登录成功（教师）
- 接口：`POST /api/auth/login`
- 请求体：`{"username":"t_zhang","password":"123456"}`
- 预期：返回 `token`、`role=TEACHER`、`userId` 非空。

### AUTH-02 登录成功（学生）
- 接口：`POST /api/auth/login`
- 请求体：`{"username":"s_wang","password":"123456"}`
- 预期：返回 `token`、`role=STUDENT`。

### AUTH-03 登录失败（密码错误）
- 请求体：`{"username":"s_wang","password":"wrong"}`
- 预期：返回错误信息（账号或密码错误）。

### AUTH-04 无 token 访问受保护接口
- 接口：`GET /api/student/overview?studentId=101`
- 预期：返回未登录/401类错误。

### AUTH-05 学生 token 访问教师接口
- 接口：`GET /api/teacher/courses`
- 预期：返回“无教师端权限”。

### AUTH-06 教师 token 访问学生接口
- 接口：`GET /api/student/overview?studentId=101`
- 预期：返回“无学生端权限”。

## 2. 教师资源归属（越权防护）测试

### OWN-01 教师获取本人课程
- 接口：`GET /api/teacher/courses`
- 预期：仅返回当前教师名下课程。

### OWN-02 教师查看非本人课程高风险（应拒绝）
- 接口：`GET /api/teacher/high-risk?courseId=<非本人课程ID>`
- 预期：返回“无权限操作非本人课程”。

### OWN-03 教师查看学生详情时带非本人课程（应拒绝）
- 接口：`GET /api/teacher/student-detail?studentId=101&courseId=<非本人课程ID>`
- 预期：拒绝访问。

### OWN-04 教师给非本人课程作业评分（应拒绝）
- 接口：`POST /api/teacher/homework-grade`
- 参数：`submissionId=<他人课程作业提交ID>`
- 预期：拒绝评分。

### OWN-05 教师查看非本人课程考试管理（应拒绝）
- 接口：`GET /api/teacher/exam-manage?courseId=<非本人课程ID>`
- 预期：拒绝访问。

## 3. 学生端业务功能测试

### STU-01 学生总览
- 接口：`GET /api/student/overview?studentId=101`
- 预期：返回 GPA、风险等级、主原因、出勤次数等字段。

### STU-02 学生课程列表
- 接口：`GET /api/student/courses?studentId=101`
- 预期：返回已选课程集合。

### STU-03 学生作业列表
- 接口：`GET /api/student/homework?studentId=101&courseId=201`
- 预期：返回作业与提交状态（submission）。

### STU-04 学生考试资格列表
- 接口：`GET /api/student/exams?studentId=101&courseId=201`
- 预期：返回 `isQualified` 与 `qualificationReason`。

### STU-05 学生风险详情
- 接口：`GET /api/student/risk-detail?studentId=101`
- 预期：返回 risk + academic。

### STU-06 学生风险趋势
- 接口：`GET /api/student/trend?studentId=101`
- 预期：返回 risk 列表、examScores、homeworkScores 等。

## 4. 作业流程测试

### HW-01 教师创建作业
- 接口：`POST /api/homework/create`
- 预期：成功落库，课程作业列表可见。

### HW-02 学生提交作业
- 接口：`POST /api/homework/submit`
- 预期：`homework_submission` 记录更新，状态为 SUBMITTED/GRADED。

### HW-03 作业提交触发风险重算
- 操作：执行 HW-02 前后对比 `risk_prediction` 最新记录。
- 预期：新增预测记录，概率/等级可变化。

### HW-04 教师评分作业
- 接口：`POST /api/teacher/homework-grade?submissionId=...&score=90&comment=...`
- 预期：提交记录分数、评语、状态更新为 GRADED。

---

## 5. 考试流程测试

### EX-01 教师创建考试
- 接口：`POST /api/exam/create`
- 预期：考试创建成功并触发资格重算。

### EX-02 学生查看考试资格（动态）
- 接口：`GET /api/student/exams?studentId=101&courseId=201`
- 预期：资格根据当前出勤动态计算，不依赖初始化值。

### EX-03 无资格提交考试（应拒绝）
- 接口：`POST /api/exam/submit`
- 预期：返回“无考试资格，后端拒绝提交”。

### EX-04 有资格提交考试
- 前置：确保学生资格为 true。
- 接口：`POST /api/exam/submit`
- 预期：`exam_record` 更新为 FINISHED，记录分数与提交时间。

### EX-05 考试提交后 GPA 与风险重算
- 操作：执行 EX-04 后检查 `student_academic` 与 `risk_prediction`。
- 预期：GPA 变化并新增风险预测记录。

---

## 6. 风险预测与预警闭环测试

### RISK-01 单学生单课程预测生成
- 触发：登录/作业/考试/GPA更新任一。
- 预期：`student_feature` 与 `risk_prediction` 新增记录。

### RISK-02 风险分层阈值
- 预期：
  - `>=0.8 => HIGH/RED`
  - `>=0.5 => MEDIUM/ORANGE`
  - `<0.5 => LOW/GREEN`

### RISK-03 预警自动生成
- 操作：触发预测后检查 `warning_record`。
- 预期：存在与 prediction 关联的预警记录，状态默认 OPEN。

### RISK-04 教师高风险筛选
- 接口：`GET /api/teacher/high-risk?courseId=...&riskLevel=HIGH&gpaColor=RED`
- 预期：结果满足筛选条件，且课程归属合法。

### RISK-05 每日定时任务全量重算（可选）
- 方式：手工触发/等待定时任务窗口。
- 预期：批量新增最新预测数据。

## 7. 干预闭环测试

### INT-01 教师创建干预记录
- 接口：`POST /api/intervention/create`（按实际参数）
- 预期：`intervention_record` 新增，关联 `warning_id` 正确。

### INT-02 干预记录查询
- 接口：`GET /api/intervention/list`（按实际参数）
- 预期：可按学生/课程查看干预历史。

### INT-03 闭环一致性核验
- 预期链路：`risk_prediction.id -> warning_record.prediction_id -> intervention_record.warning_id`。

## 8. 数据一致性与边界场景

### DATA-01 重复提交幂等性
- 作业、考试重复提交时是否覆盖/拒绝符合预期。

### DATA-02 空课程学生预测
- 无选课学生是否走“student级预测（courseId=null）”分支。

### DATA-03 极端数据
- 0出勤、0作业、缺考情况下，系统是否稳定返回高风险结果。

### DATA-04 分页与性能（如有分页接口）
- 大数据量下列表接口是否响应正常。

## 10. 验收判定标准
- P0（必须通过）：鉴权、角色隔离、教师归属校验、考试资格拒绝、风险写库、预警写库。
- P1（应通过）：学生端核心查询、作业/考试流程、干预创建与查询。
- P2（可迭代）：性能压测、批量任务时效、复杂边界场景。

当 P0 全通过且 P1 ≥ 95% 通过，可判定“后端功能可用于答辩演示”。
